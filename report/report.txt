- fileHandler - which is responsible for loading to and from the transaction log file, it includes 2 functions

save_transactions() - checks if the file exists, if it does, it overwrites the entire file and goes node by node and writes the contents of it on each line
load_transactions() - checks if the file exists, making one if it doesnt, then it checks each line for up to 99 characters followed by a '|' then a floating point number followed by another '|'and then either 99 more characters or the end of the line, if it finds a line of this description, it loads each of the three into variables and then to a new node of the linked list, then it moves on to the next line

- linkedListHander - which is responsible for adding, removing and printing the list of transactions logically in the linked list, it has 7 functions

create_transaction_node() - creates a new node in the data structure, checks if its null and then loads the description, amount, status and then sets the next node to null
count_transactions() - makes an int called count and then iterates through the linked list until it finds a null node and then returns the count
insert_node_at_position() - checks if the node is null, then if the node is being set inserted at position 1, it sets the current head to the next node and makes the new node the new head. if its not at position 1, it increases the count and makes a new node after the position and changes the links before and after to apply to the new node

delete_transaction() - checks if the list is empty, and the calls the print transaction function so that the user can see what they want to delete, then if the position for deletion is 1, it allocates the next node as the new head, and deletes the current head, otherwise it travels through the linked list until it finds the correct position and deletes it, relinking the node before and after it to connect to each other
